### v1.0.2 — 并发合并、稳定性增强与打包优化

此次更新在原有并发优化的基础上，引入了更深层的翻译管理机制，显著提升了高并发场景下的稳健性与执行效率。

**✨ 新增 (Added)**
- **并发请求合并 (Coalescing)**：针对相同原文的多个并发请求，现在会智能合并为单个 API 调用，极大减少重复翻译的开销与 API 消耗。
- **实时并发调节**：引入动态信号量（Adjustable Semaphore），支持在不重启服务的情况下实时调整最大并发数。
- **智能跳过逻辑**：
    - **长度过载保护**：自动识别并跳过超过 token 限制的超长文本，避免必然的请求超时。
    - **短文本智能穿透**：针对长度 <= 10 且主要由汉字或符号组成的文本（如角色名、符号组合）增加直通逻辑，提升整体翻译通吐。

**⚡ 性能与反馈 (Performance & UX)**
- **动态连接池**：HTTP 连接池容量随并发数自动缩放，彻底消除高并发下的连接争用。
- **退避重试策略**：重试机制加入指数退避（Exponential Backoff）与随机抖动（Jitter），显著降低后端过载风险。
- **检测算法优化**：重写了文本重复检测逻辑，将嵌套循环优化为线性检查，处理速度更快。

**🛡️ 稳健性 (Stability)**
- **配置原子化保存**：优化 `ConfigStore`，通过临时文件转正机制确保配置保存时即使掉电也不会损坏 `config.json`。
- **生命周期管理**：优化了服务停止流程，支持强制终止卡死的线程。
- **错误状态治理**：移除了带有竞态问题的端口占用预检，统一由网络层异常捕获处理。

**📦 打包与流水线 (Build & CI)**
- **体积缩减 [FIXED]**：已正确应用 `.spec` 指令，排除了 numpy/pandas 等大型库，体积降至 50-70MB。
- **CI 加速**：GitHub Actions 现已支持 Pip 缓存与并发构建合并（Auto-cancel），构建速度显著提升。
